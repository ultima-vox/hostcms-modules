# Optimizer для HostCMS 7

Пакет модуля серверной оптимизации HTML-страниц для HostCMS 7.

Модуль выполняет постобработку готового HTML-ответа и позволяет по отдельности включать необходимые функции. Все преобразования отключены по умолчанию.

## Возможности

- минификация HTML;
- удаление безопасных HTML-комментариев;
- объединение и минификация локальных CSS-файлов;
- объединение и минификация локальных JavaScript-файлов;
- добавление `loading="lazy"` и `decoding="async"` изображениям;
- исключение первых изображений и изображений с заданными CSS-классами;
- подмена JPG и PNG на существующие WebP или AVIF;
- серверная генерация WebP и AVIF через GD или Imagick;
- пакетное сканирование каталогов через AJAX;
- DNS Prefetch и Preconnect;
- предзагрузка шрифтов;
- вставка Critical CSS.

optimizer/

Каталоги `admin` и `modules` внутри пакета соответствуют каталогам в корне установленной HostCMS.

## Установка

1. Скопируйте содержимое каталога `optimizer/admin/` в фактический каталог центра администрирования HostCMS.
2. Скопируйте содержимое каталога `optimizer/modules/` в `/modules/` сайта.
3. В центре администрирования HostCMS откройте раздел управления модулями.
4. Добавьте модуль с кодом `optimizer`.
5. Выполните установку и активируйте модуль.
6. Откройте пункт `Optimizer` в меню центра администрирования.

После установки модуль создаёт каталог:

```text
/upload/optimizer_cache/
```

PHP-процесс должен иметь право записи в `/upload/`.

## Обновление

При обновлении замените файлы:

```text
/admin/optimizer/
/modules/optimizer/
```

Настройки хранятся отдельно в:

```text
/upload/optimizer_cache/settings_<site_id>.json
```

Замена файлов модуля не удаляет пользовательские настройки.

## Генерация WebP и AVIF

Во вкладке «Изображения» можно:

- проверить поддержку WebP и AVIF на сервере;
- выбрать форматы генерации;
- задать качество;
- указать каталоги для сканирования относительно корня сайта;
- задать лимит файлов за один AJAX-проход;
- ограничить максимальный размер исходного изображения.

Пример каталогов:

```text
upload
images
upload/information_system_1
```

Указывайте по одному каталогу в строке.

Генератор:

- обрабатывает JPG, JPEG и PNG;
- создаёт файлы рядом с исходниками;
- не пересоздаёт актуальные файлы;
- пересоздаёт вариант, если исходник был изменён позже;
- не запускается при обычном открытии страниц сайта.

## Безопасность

Модуль не обрабатывает:

- административную часть;
- запросы к `/hostcmsfiles/` и `/modules/`;
- CLI-запуски;
- POST-запросы;
- AJAX и JSON-ответы;
- XML, RSS, sitemap и другие не-HTML ответы;
- неполные HTML-фрагменты.

Для исключения отдельного изображения добавьте атрибут:

```html
<img src="/images/example.jpg" data-optimizer-skip>
```

Либо добавьте CSS-класс изображения в список исключений в настройках модуля.

## Рекомендации

Включайте функции по одной и проверяйте:

- главную страницу;
- карточки товаров;
- формы;
- корзину и оформление заказа;
- личный кабинет;
- страницы с внешними скриптами;
- адаптивную версию сайта.

Объединение CSS и JavaScript может быть несовместимо с отдельными шаблонами и сторонними библиотеками. Используйте эти функции только после проверки сайта.

## Удаление

Метод `uninstall()` удаляет каталог:

```text
/upload/optimizer_cache/
```

вместе с настройками и кэшированными файлами модуля.

Сгенерированные WebP и AVIF, расположенные рядом с исходными изображениями, автоматически не удаляются.
