# Page Optimizer — модуль оптимизации страниц для HostCMS 7

Модуль серверной постобработки HTML-страниц для HostCMS 7. Все преобразования выключены по умолчанию и включаются отдельно в административной части модуля.

## Текущие возможности

- безопасная проверка контекста ответа до обработки;
- консервативная минификация HTML;
- опциональное удаление безопасных HTML-комментариев;
- добавление `loading="lazy"` для изображений;
- подмена локальных изображений на существующие WebP/AVIF-файлы;
- DNS Prefetch, Preconnect и preload шрифтов;
- вставка Critical CSS;
- экспериментальное объединение и минификация CSS/JS;
- хранение настроек отдельно для каждого сайта;
- административная панель настроек и базовой статистики.

## Структура

```text
admin/page_optimizer/index.php
modules/page_optimizer/module.php
modules/page_optimizer/PageOptimizer.php
modules/page_optimizer/PageOptimizer_Context.php
modules/page_optimizer/PageOptimizer_Html.php
modules/page_optimizer/PageOptimizer_Settings.php
modules/page_optimizer/PageOptimizer_Assets.php
modules/page_optimizer/i18n/ru.php
```

Классы модуля загружаются из `module.php`. Для совместимости с различными сборками HostCMS объявляются оба варианта имени основного класса модуля:

```php
Page_Optimizer_Module
PageOptimizer_Module
```

## Требования

- HostCMS 7;
- PHP-версия, поддерживаемая установленной версией HostCMS;
- право PHP-процесса создавать каталог и файлы внутри `/upload/`;
- административная директория HostCMS должна содержать каталог `page_optimizer`.

## Установка

1. Скопируйте каталог:

   ```text
   modules/page_optimizer
   ```

   в каталог модулей HostCMS:

   ```text
   /modules/
   ```

2. Скопируйте каталог:

   ```text
   admin/page_optimizer
   ```

   в текущий каталог центра администрирования HostCMS. Если административная директория была переименована, используйте её фактическое имя вместо `/admin/`.

3. Убедитесь, что PHP-процесс может записывать в:

   ```text
   /upload/
   ```

4. В центре администрирования откройте раздел **Модули** и добавьте модуль с кодом:

   ```text
   page_optimizer
   ```

5. Выполните установку модуля. Метод `install()` создаст:

   ```text
   /upload/page_optimizer_cache/
   ```

   и начальный файл безопасных настроек.

6. Активируйте модуль.

7. Откройте пункт **Page Optimizer** в меню центра администрирования.

После установки все преобразования выключены. Сначала рекомендуется включить только минификацию HTML и проверить основные типы страниц сайта.

## Обновление

При обновлении замените файлы в каталогах:

```text
/modules/page_optimizer/
/<admin-directory>/page_optimizer/
```

Существующие настройки сохраняются. При изменении версии конфигурации устаревшие агрессивные параметры автоматически переводятся в безопасное состояние.

## Удаление

Метод `uninstall()` удаляет каталог:

```text
/upload/page_optimizer_cache/
```

вместе с настройками и сгенерированными файлами модуля.

## Диагностика установки

Если HostCMS не добавляет или не устанавливает модуль:

1. проверьте журнал HostCMS в `/hostcmsfiles/logs/`;
2. проверьте наличие `/modules/page_optimizer/module.php`;
3. проверьте права записи в `/upload/`;
4. проверьте, что код модуля указан как `page_optimizer`;
5. временно оставьте модуль неактивным до устранения ошибки.
